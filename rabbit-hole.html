RAD-PDSA release path

Follow the Asymmetry.

The first page is the door. This page is the map.

RAD-PDSA is not one claim. It is a stack: stability architecture, runtime tests, mathematical structure, physical interpretation, and proposed hardware embodiments.

The public path

  1. Start with the stability engine.
  2. Review the runtime behavior and ablation tests.
  3. Study the zero-state and measure-zero arguments.
  4. Move into the toroidal geometry and scale-invariant model.
  5. Only then approach the cosmology and Big Bang interpretation.

Claim discipline

Every claim must be labeled by status: theorem, implementation, simulation result, proposed testbed, or speculative extension. This protects the work from being judged as a single oversized assertion.

Core release target

The first serious paper should be the Technical Validation Report for the v7 RAD Engine: a sober report showing how the engine behaves under perturbation, ablation, recovery, and bounded-corridor enforcement.
